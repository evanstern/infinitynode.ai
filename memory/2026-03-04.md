### Venice model discovery blocking event loop — FIXED (workaround)
- **Root cause:** Venice model discovery (`venice-models.ts`) blocks the Node.js event loop for ~38 seconds during RPC handling. This froze all WebSocket connections, killed Slack sockets, and caused cascading gateway restarts (11+ in a single day).
- **Why it persisted after config removal:** Agent-level cached files (`~/.openclaw/agents/main/agent/auth-profiles.json` and `models.json`) retained Venice config even after removing it from `openclaw.json` and `.env`.
- **Fix applied:**
  1. Removed Venice provider from `openclaw.json`
  2. Removed `VENICE_API_KEY` from `~/.openclaw/.env`
  3. Purged Venice from cached `auth-profiles.json` and `models.json`
  4. Fixed `claude-opus-4.6` typo (dot → dash) in config
- **Result:** `config.get` RPC dropped from 38,325ms → 936ms
- **Remaining:** Venice discovery still fires on startup (hardcoded in binary) but no longer blocks runtime RPCs
- **Bug filed:** https://github.com/openclaw/openclaw/issues/34914
